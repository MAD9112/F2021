# 1. iOS App

Currently there are 2 ways of building apps for iOS in the context of UI creation and use: declarative and imperative.

# Declarative UI

A declarative UI, basically, describes what elements are part of the user interface. Some of the details can be left out, e.g,: style and exact position of the elements. SwiftUI is a declarative way to build interfaces in iOS apps. It is easy to learn, clean and provides simple tools to implement reactive programming. There are a drawback though: It is supported only on iOS13 and above. So, if you want your app to run in an older iOS version, SwiftUI may not be the best option for you.

# Imperative UI

A imperative UI, by contrast has to explicit define all details on style and position. UIKit is the library we use to create imperative UI.

Question

So, what should you learn and what should I use?

Another distinction that you should be familiar with is the one between Reactive and Imperative programmimg.

Imperative programs use statements to change the app's state. One of the most used patterns used in Imperative programming is the Model-View-Controller.

Reactive programs, differently, uses data streams, propagation of change and bindings.
